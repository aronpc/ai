# AI Attribution Scrubber

Antes de permitir um `git commit`, verifique se a mensagem ou os metadados contem atribuicao a AI. Se encontrar, BLOQUEIE o commit.

## Padroes a Detectar

### Na Mensagem de Commit

BLOQUEIE se a mensagem contem (case-insensitive):

- `Generated by Claude`
- `Generated with Claude`
- `Created by Claude`
- `Written by Claude`
- `Claude AI`
- `Anthropic`
- `AI-generated`
- `AI-assisted`
- `Co-authored-by: Claude`
- `Co-Authored-By:.*claude`
- `Co-Authored-By:.*anthropic`
- `Co-Authored-By:.*noreply@anthropic`
- `Co-authored-by:.*copilot`
- `Co-authored-by:.*github.com`
- `🤖` (emoji de robo no commit message)

### Nos Trailers do Commit

BLOQUEIE se o comando `git commit` inclui:
- `--trailer "Co-authored-by:.*"`
- `-m` com qualquer padrao acima

## Acao ao Detectar

Se atribuicao AI for detectada:

1. **BLOQUEIE** o commit
2. Mostre a mensagem limpa sugerida (removendo linhas com atribuicao)
3. Instrua:
   ```
   Commit bloqueado: contem atribuicao AI.
   Commits devem parecer escritos por um desenvolvedor humano.
   Mensagem limpa sugerida:
   [mensagem sem linhas de atribuicao]
   ```

## Verificacoes Adicionais

### Conventional Commits Format

Verifique se a mensagem segue o formato:
```
tipo(escopo): descricao

corpo opcional

footer opcional
```

Tipos validos: `feat`, `fix`, `refactor`, `docs`, `test`, `chore`, `style`, `perf`, `ci`, `build`

Se NAO seguir, **AVISE** (nao bloqueie):
```
Aviso: mensagem de commit nao segue Conventional Commits.
Formato: tipo: descricao (ex: feat: add user authentication)
```

## Comportamento

- Atribuicao AI: BLOQUEIE (hard block)
- Conventional Commits: AVISE (soft warning)
- Aplique a TODOS os comandos git commit
- Verifique tanto `-m "mensagem"` quanto heredoc/EOF patterns
