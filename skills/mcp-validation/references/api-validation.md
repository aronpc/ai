# Referência de Validação de API

Validação REST/GraphQL.

## Passos

1. Verificar endpoints existentes

3. Verificar tratamento de erros

## Códigos de Status

| Código | Significado |
|--------|-------------|
| 200 | OK |
| 201 | Criado |
| 400 | Requisição inválida |
| 401 | Não autorizado |
| 404 | Não encontrado |
| 500 | Erro do servidor |
